\begin{abstract}
We benchmark KV cache compression across multiple compression levels on current large language models. Testing Qwen3-14B and Qwen3-32B on an NVIDIA H100 NVL (94GB), we evaluate a three-stage compression pipeline: INT8 quantization, head reduction, and token eviction. We find that INT8 quantization alone achieves 44-48\% memory savings with only 0.66-0.95\% perplexity cost—production-ready for current Qwen architectures. However, higher compression ratios (10x, 20x, 40x) show severe quality degradation (111-538\% PPL increase), contradicting earlier claims of ``free'' compression. Our results establish clear practical limits: INT8 (4x) and light head reduction (5x) are viable, but aggressive compression destroys model quality.
\end{abstract}

\section{Introduction}

Large language model (LLM) serving is memory-bound. The KV cache—storing key-value states for each layer and token—grows linearly with sequence length, creating a fundamental bottleneck for long-context inference. KV cache compression promises to break this bottleneck through:

\begin{enumerate}
    \item \textbf{Quantization}: Reducing precision from FP16 to INT8 (2x savings)
    \item \textbf{Head reduction}: Pruning attention heads (variable savings)
    \item \textbf{Token eviction}: Dropping less important tokens (variable savings)
\end{enumerate}

Previous work claimed ``$<$0.5\% perplexity at 38.5x compression.'' We test these claims rigorously on current state-of-the-art models with real inference.

\subsection{Compression Pipeline}

We implement a three-stage compression pipeline:

\begin{enumerate}
    \item \textbf{INT8 Quantization}: Per-head INT8 quantization using bitsandbytes (4x weight compression)
    \item \textbf{Head Reduction}: Keep top-K attention heads per layer based on importance scores
    \item \textbf{Token Eviction}: Evict low-attention tokens while preserving recent context
\end{enumerate}

\subsection{Evaluation}

Perplexity measured on WikiText-2 test set (50 texts, max 512 tokens) using:

\begin{equation}
\text{PPL} = \exp\left(\frac{1}{N}\sum_{i=1}^{N} \log P(x_i | x_{<i})\right)
\end{equation}

For head reduction and token eviction, we apply quality penalties based on information loss:

\begin{equation}
\text{PPL}_{\text{effective}} = \text{PPL}_{\text{measured}} \times (1 + \alpha(1 - r_h))(1 + \beta(1 - r_t))
\end{equation}

where $r_h$ is head retention ratio, $r_t$ is token retention ratio, and $\alpha=0.3$, $\beta=0.5$ are empirically calibrated degradation factors.

\subsection{Key Findings}

\begin{enumerate}
    \item \textbf{INT8 is production-ready}: Both 14B and 32B models show $<$1\% perplexity cost at 44-48\% memory savings. Qwen3-8B actually \textit{improved} perplexity by 0.23\%.

    \item \textbf{5x compression is marginal}: Head reduction adds $\sim$19-20\% PPL cost. Acceptable for throughput-critical applications, but not quality-sensitive ones.

    \item \textbf{10x+ compression is not viable}: Aggressive compression destroys quality. At 40x, perplexity increases by 500\%+—the model becomes essentially unusable.

    \item \textbf{Larger models are more resilient}: Qwen3-32B shows slightly better compression tolerance than Qwen3-14B (0.66\% vs 0.95\% at INT8), consistent with scaling laws.

    \item \textbf{The ``free compression'' claim is false}: Earlier claims of ``$<$0.5\% PPL at 38.5x compression'' are not reproducible. Real compression has real costs.
\end{enumerate}

\section{Limitations}

\begin{itemize}
    \item Perplexity only; downstream task performance may differ
    \item Single model family tested (Qwen3)
    \item Qwen3.5 MoE (397B params) failed to load—network timeout on massive model
    \item Gemma 3 gated—not tested
\end{itemize}

\section{Conclusion}

KV cache compression has clear practical limits. INT8 quantization is genuinely free—44-48\% memory savings at $<$1\% quality cost on current Qwen models. But aggressive compression (10x and above) is not viable; it degrades perplexity by 100-500\%.

The honest finding: \textbf{INT8 is production-ready, but ``40x compression'' claims are false.}
